# Rebis → Python Transpilation Showcase

A side-by-side walkthrough of what a Rebis DSL file looks like and what the transpiler
generates for the Python target.

---

## The app: Todo List

Features demonstrated:
- Component with parameters (`todo_item`)
- Reactive state (`state items: string[]`, `state input: String`)
- Data binding (`bind=state.input`)
- Event handlers with imperative logic (`on add`, `on toggle`, `on delete`)
- List rendering from state
- Conditional visibility
- Platform overrides

---

## Rebis source (`src/todo.rebis`)

```rebis
import { format_date } from "./utils"

export component todo_list {

    // --- State ---
    state items: string[] = []
    state input = ""
    state edit_index: int? = null

    // --- UI ---
    column padding=16 spacing=12
        // Header
        text value="Tasks for ${format_date(today())}" font="bold 20"

        // Input row
        row spacing=8
            text_input new_task placeholder="What needs doing?" bind=state.input
            button label="Add" width=80 action=add
            button label="Clear" width=60 action=clear
                visible=state.input != ""

        // Item count
        text value="${state.items.len()} items" color="#888" font="12"

        // List of items
        row spacing=8 padding=8
            @platform(web) {
                background = #f5f5f5
            }
            column spacing=4
                for item in state.items
                    checkbox label=item.text checked=item.done action=on_toggle(item.id)
                        visible=true
                text value="Nothing yet — add a task above"
                    visible=state.items.len() == 0

    // --- Event handlers ---
    on add {
        let text = state.input.trim()
        if text != "" {
            state.items.push(text)
            state.input = ""
        }
    }

    on clear {
        state.input = ""
    }

    on toggle { id ->
        let item = state.items.get(id)
        if item {
            item.done = !item.done
        }
    }

    on delete { id ->
        state.items.remove(id)
    }
}
```

## Generated Python (`generated/todo.py`)

What the transpiler emits for the `python` target. All imports, classes, and event wiring
are generated automatically.

```python
"""
Auto-generated by Rebis transpiler (target: python/tkinter)
Source: src/todo.rebis
Generated: 2026-07-01T12:00:00Z
Template mapping:
  - row          → tk.Frame(side=tk.TOP, ...)
  - column       → tk.Frame(side=tk.TOP, ...)
  - text         → tk.Label(...)
  - text_input   → tk.Entry(...)
  - button       → tk.Button(...)
  - checkbox     → tk.Checkbutton(...)
  - spacer       → tk.Frame(height/width)
  - scroll_view  → tk.Canvas + scrollbar
"""
import tkinter as tk
from tkinter import ttk
from typing import Optional, List
from dataclasses import dataclass, field
from datetime import date


# ─── Transpiled from ./utils ─────────────────────────────────────────────
def format_date(d: date) -> str:
    """Rebis source: format_date"""
    return d.strftime("%A, %B %d")


def today() -> date:
    """Rebis source: today"""
    return date.today()


# ─── Generated components ─────────────────────────────────────────────────


@dataclass
class TodoItem:
    """Model for items in the todo list."""
    id: int
    text: str
    done: bool = False


class TodoList(tk.Frame):
    """
    Rebis component: todo_list
    File: src/todo.rebis:4
    """

    def __init__(self, master=None, **kwargs):
        super().__init__(master, **kwargs)
        self.pack(fill=tk.BOTH, expand=True)

        # ── Reactive state ────────────────────────────────────────────────
        self._state = {
            "items": [],     # List[TodoItem]
            "input": "",     # str
            "edit_index": None,
        }
        self._item_counter = 0

        # ══════════════════════════════════════════════════════════════════
        # UI tree — column padding=16 spacing=12
        # ══════════════════════════════════════════════════════════════════
        outer = tk.Frame(self)
        outer.pack(fill=tk.BOTH, expand=True, padx=16, pady=16)

        # ── text value="Tasks for {format_date(today())}" font="bold 20" ──
        header_text = "Tasks for " + format_date(today())
        header = tk.Label(
            outer,
            text=header_text,
            font=("TkDefaultFont", 20, "bold"),
            anchor="w",
        )
        header.pack(fill=tk.X)

        # ── row spacing=8 ─────────────────────────────────────────────────
        input_row = tk.Frame(outer)
        input_row.pack(fill=tk.X, pady=(8, 0))

        # ── text_input new_task placeholder="What needs doing?" bind=state.input ──
        self.input_var = tk.StringVar()
        input_field = tk.Entry(
            input_row,
            textvariable=self.input_var,
        )
        input_field.pack(side=tk.LEFT, fill=tk.X, expand=True)

        # ── button label="Add" width=80 action=add ────────────────────────
        add_btn = tk.Button(
            input_row,
            text="Add",
            width=8,
            command=self._on_add,
        )
        add_btn.pack(side=tk.LEFT, padx=(8, 0))

        # ── button label="Clear" width=60 action=clear
        #    visible=state.input != "" ──────────────────────────────────────
        self.clear_btn = tk.Button(
            input_row,
            text="Clear",
            width=6,
            command=self._on_clear,
        )
        # visible binding — hidden when input is empty
        self.clear_btn.pack(side=tk.LEFT, padx=(8, 0))

        # ── text value="{state.items.len()} items" color="#888" font="12" ──
        self.item_count = tk.Label(
            outer,
            text="0 items",
            font=("TkDefaultFont", 12),
            foreground="#888888",
            anchor="w",
        )
        self.item_count.pack(fill=tk.X, pady=(4, 0))

        # ── row spacing=8 padding=8 ───────────────────────────────────────
        list_frame = tk.Frame(outer)
        list_frame.pack(fill=tk.BOTH, expand=True, pady=(8, 0))

        # ── column spacing=4 ──────────────────────────────────────────────
        self.list_container = tk.Frame(list_frame)
        self.list_container.pack(fill=tk.BOTH, expand=True)

        # ── text value="Nothing yet — add a task above"
        #    visible=state.items.len() == 0 ─────────────────────────────────
        self.empty_label = tk.Label(
            self.list_container,
            text="Nothing yet — add a task above",
            font=("TkDefaultFont", 10),
            foreground="#999999",
        )
        self.empty_label.pack()

        # ── State → UI sync ───────────────────────────────────────────────
        self._sync()

        # ── Bind input changes to state.input ─────────────────────────────
        self.input_var.trace_add("write", self._on_input_changed)

    # ─── Event: on add ────────────────────────────────────────────────────
    def _on_add(self):
        """Rebis source: src/todo.rebis:36 (on add)"""
        text = self.input_var.get().strip()
        if text != "":
            self._state["input"] = ""
            self.input_var.set("")
            item = TodoItem(id=self._item_counter, text=text, done=False)
            self._state["items"].append(item)
            self._item_counter += 1
            self._sync()

    # ─── Event: on clear ──────────────────────────────────────────────────
    def _on_clear(self):
        """Rebis source: src/todo.rebis:43 (on clear)"""
        self._state["input"] = ""
        self.input_var.set("")
        self._sync_clear_btn()

    # ─── Event: on toggle ─────────────────────────────────────────────────
    def _on_toggle(self, item_id: int):
        """Rebis source: src/todo.rebis:47 (on toggle)"""
        for item in self._state["items"]:
            if item.id == item_id:
                item.done = not item.done
                break
        self._sync_list()

    # ─── Event: on delete ─────────────────────────────────────────────────
    def _on_delete(self, item_id: int):
        """Rebis source: src/todo.rebis:53 (on delete)"""
        self._state["items"] = [
            item for item in self._state["items"] if item.id != item_id
        ]
        self._sync()

    # ─── Input binding callback ───────────────────────────────────────────
    def _on_input_changed(self, *_):
        self._state["input"] = self.input_var.get()
        self._sync_clear_btn()

    # ─── State → UI sync ──────────────────────────────────────────────────
    def _sync(self):
        """Sync entire UI tree with current state."""
        self._sync_list()
        self._sync_item_count()
        self._sync_clear_btn()

    def _sync_list(self):
        """Rebis: for item in state.items"""
        # Clear existing item widgets
        for child in self.list_container.winfo_children():
            child.destroy()

        if len(self._state["items"]) == 0:
            # visible=state.items.len() == 0 — show empty state
            self.empty_label = tk.Label(
                self.list_container,
                text="Nothing yet — add a task above",
                font=("TkDefaultFont", 10),
                foreground="#999999",
            )
            self.empty_label.pack()
        else:
            for item in self._state["items"]:
                self._render_todo_item(item)

    def _render_todo_item(self, item: TodoItem):
        """Rebis: checkbox label=item.text checked=item.done ..."""
        row = tk.Frame(self.list_container)
        row.pack(fill=tk.X, pady=2)

        checked = tk.BooleanVar(value=item.done)
        cb = tk.Checkbutton(
            row,
            text=item.text,
            variable=checked,
            command=lambda i=item.id: self._on_toggle(i),
        )
        cb.pack(side=tk.LEFT)

        delete_btn = tk.Button(
            row,
            text="✕",
            width=2,
            command=lambda i=item.id: self._on_delete(i),
        )
        delete_btn.pack(side=tk.RIGHT)

    def _sync_item_count(self):
        """Update item count label."""
        count = len(self._state["items"])
        self.item_count.config(text=f"{count} items")

    def _sync_clear_btn(self):
        """Rebis: visible=state.input != ''"""
        if self._state["input"] != "":
            self.clear_btn.pack(side=tk.LEFT, padx=(8, 0))  # show
        else:
            self.clear_btn.pack_forget()  # hide


# ─── Entry point ──────────────────────────────────────────────────────────
if __name__ == "__main__":
    root = tk.Tk()
    root.title("Todo List — Rebis → Python")
    root.geometry("480x520")
    app = TodoList(root)
    root.mainloop()
```

## What the transpiler did, step by step

### 1. Component → Class
```
Rebis:  component todo_list { ... }
Python: class TodoList(tk.Frame): ...
```

Every component becomes a class. Layout components map to `tk.Frame`. Leaf
components map to specific widgets.

### 2. State → Instance dict
```
Rebis:  state items: string[] = []
        state input: String = ""
Python: self._state = {"items": [], "input": ""}
```

State becomes a dictionary with typed values. Mutations trigger `_sync()`.
The transpiler generates a diff-based sync for each bound property.

### 3. Tag tree → Widget tree

Each nested tag becomes a `pack()` or `grid()` layout call. The transpiler
infers geometry management from the parent container type:

```
Rebis:          Python:
row             tk.Frame → side=tk.LEFT
column          tk.Frame → side=tk.TOP
stack           tk.Frame → place() / geometry
scroll_view     tk.Frame + tk.Scrollbar + tk.Canvas
```

### 4. Properties → Widget config

```
Rebis:  text value="Tasks for ..." font="bold 20"
Python: tk.Label(text="Tasks for ...", font=(..., 20, "bold"))
```

The transpiler knows the property-to-constructor mapping for every built-in
component per target. Platform-specific properties (like `secure=true` →
`show="*"` on Entry) are handled by the template engine.

### 5. Bindings → trace / observable

```
Rebis:  text_input new_task bind=state.input
Python: self.input_var = tk.StringVar()
        self.input_var.trace_add("write", self._on_input_changed)
```

Two-way bindings generate a `tk.StringVar` / `tk.BooleanVar` and wire a
change listener that syncs back to the state dict. One-way bindings
(`bind_in`) only update the widget; `bind_out` only updates the state.

### 6. Event handlers → methods

```
Rebis:  on add {
            let text = state.input.trim()
            if text != "" {
                state.items.push(text)
                state.input = ""
            }
        }
Python: def _on_add(self):
            text = self._state["input"].strip()
            if text != "":
                self._state["items"].append(...)
                self._state["input"] = ""
                self._sync()
```

Each `on name { }` block becomes a private method. C-family control flow
(`if`, `for`, `let`, `try/catch`) maps directly to Python equivalents.
State mutations are followed by a `_sync()` call that updates the UI tree.

### 7. For loops → widget factories

```
Rebis:  for item in state.items
            checkbox label=item.text ...
Python: for item in self._state["items"]:
            self._render_todo_item(item)
```

List rendering generates a factory method that creates and packs child widgets
for each element. On state change, the entire list section is rebuilt (or,
with the advanced backend, diffed and patched in place).

### 8. Platform overrides → conditional code

```
Rebis:  @platform(web) { background = #f5f5f5 }
Python: (ignored for python target — platform block is for web only)
```

The transpiler evaluates `@platform(name)` at generation time. Blocks for
other targets are stripped; blocks for the current target are inlined.

## What changes for other targets?

Each target backend swaps the component → widget mapping table:

| Target | Layout | Button | Text Input | State |
|---|---|---|---|---|
| Python/tkinter | `tk.Frame` | `tk.Button` | `tk.Entry` | `dict` + `_sync()` |
| Web/React | `<div>` | `<button>` | `<input>` | `useState()` |
| iOS/SwiftUI | `VStack` | `Button` | `TextField` | `@State` |
| Android/Kotlin | `Column` | `Button` | `TextField` | `mutableStateOf` |
| C++/Qt | `QVBoxLayout` | `QPushButton` | `QLineEdit` | signals + slots |

The DSL source stays the same. Only the template set changes.
